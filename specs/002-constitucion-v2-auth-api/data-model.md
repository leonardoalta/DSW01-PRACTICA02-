# Data Model - Ajuste a Constitución v2.1 (PK autogenerada)

## Entity: Empleado

### Fields
- `clave` (string, required, unique, PK, autogenerated)
  - Regla: se genera automáticamente al crear empleado; no se recibe desde API de alta.
  - Uso: llave primaria de persistencia e identificador canónico de autenticación.
- `nombre` (string, required, maxLength: 100)
  - Regla: no nulo, longitud entre 1 y 100.
- `telefono` (string, required, maxLength: 100)
  - Regla: no nulo, longitud entre 1 y 100.
- `contrasenaHash` (string, required)
  - Regla: hash BCrypt; nunca retornar en respuestas API.
- `activo` (boolean, required, default: true)
  - Regla: solo empleados activos pueden autenticarse y aparecer en consultas operativas.
- `createdAt` (datetime, required)
- `updatedAt` (datetime, required)

### Relationships
- No requiere relaciones adicionales en este alcance.

### Validation Rules
- Unicidad de `clave` obligatoria y garantizada por mecanismo de generación backend + restricción de base de datos.
- `nombre` y `telefono` rechazan valores con longitud > 100.
- `contrasenaHash` debe representar un hash válido de contraseña.
- Operaciones de actualización/eliminación requieren existencia por `clave`.
- Eliminación lógica sobre empleado inactivo retorna conflicto de estado.
- Si `page` se envía en listado, debe ser entero >= 0.
- Tamaño de página es fijo de 5 en modo paginado.

### State Transitions
- `ACTIVO` (`activo=true`) → `INACTIVO` (`activo=false`) al eliminar lógicamente.
- `INACTIVO` no puede autenticarse ni exponerse en consultas operativas.
